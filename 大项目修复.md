# 大型 C/C++ 项目中文乱码修复说明

本文档用于处理以下场景：

- 远程仓库网页端中文已经乱码；
- 本地重新 clone 后中文仍然乱码；
- 项目较大，部分文件正常，部分文件乱码；
- 乱码可能出现在 `.c/.cpp/.h/.hpp/.txt/.md/.cmake` 等文本文件中；
- 希望尽可能批量修复，但又避免误伤正常文件。

---

## 1. 问题原因

Git 本身通常不会主动破坏中文内容。
如果远程仓库网页端已经乱码，说明当前仓库中保存的文件内容本身大概率已经是乱码。

常见原因包括：

1. 原文件是 UTF-8，但被错误地按 GBK 打开；
2. 打开后看到乱码，又直接保存；
3. 乱码文本被重新保存成 UTF-8；
4. 乱码文件被提交到 Git；
5. 之后无论远程网页端还是本地 clone，看到的都是乱码。

此时问题已经不是简单的“换 GBK/UTF-8 打开”能解决的，而需要：

- 从历史版本或备份恢复；
- 或者对可逆乱码进行反向修复；
- 或者人工重写已经不可逆损坏的中文内容。

---

## 2. 修复前必须注意

不要直接在原项目上批量覆盖。

推荐先新建修复分支：

```bash
git switch -c fix/chinese-encoding
```

也可以先完整备份项目：

```bash
cp -r project project_backup
```

或者在 Windows 下直接复制一份项目目录。

---

## 3. 乱码是否一定能修复？

不一定。

### 3.1 大概率可以修复的情况

如果乱码类似：

```text
涓枃
鏂囦欢
浠ｇ爜
```

或者：

```text
ä¸­æ–‡
æ–‡ä»¶
```

这类通常是编码被错误解释后保存，部分情况下可以反向修复。

### 3.2 很难完整修复的情况

如果文件里大量出现：

```text
���
```

或者：

```text
?????
```

这通常说明原始字节已经被替换字符覆盖，信息已经丢失。
这种情况下脚本无法完整恢复，只能从 Git 历史、备份、队友本地代码、IDE Local History 或人工重写恢复。

---

## 4. 推荐修复流程

整体流程如下：

```text
1. 新建修复分支
2. 运行扫描脚本，生成 report
3. 查看哪些文件疑似乱码
4. 查看候选修复文件
5. 人工确认候选文件是否恢复正常
6. 确认后再自动覆盖或手动替换
7. 重新 build/debug/test
8. 提交修复结果
9. 加入编码规范，避免再次发生
```

---

## 5. 新建修复脚本

在项目中新建目录：

```bash
mkdir -p tools
```

新建文件：

```text
tools/fix_chinese_encoding.py
```

内容如下：

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-

"""
大型项目中文乱码扫描与候选修复工具。

默认行为：
1. 扫描项目中的文本文件；
2. 跳过 .git、build、cmake-build-*、node_modules 等目录；
3. 判断文件是否可能存在中文乱码；
4. 对疑似乱码文件生成候选修复文件；
5. 输出 CSV 报告；
6. 默认不覆盖原文件。

典型用法：

    python tools/fix_chinese_encoding.py --root .

生成候选文件后，人工检查 .encoding_fix_candidates 目录和 encoding_fix_report.csv。

确认后自动覆盖：

    python tools/fix_chinese_encoding.py --root . --apply

更激进地尝试二次反乱码：

    python tools/fix_chinese_encoding.py --root . --max-depth 2
"""

from __future__ import annotations

import argparse
import csv
import re
import shutil
from dataclasses import dataclass
from pathlib import Path
from typing import Callable, Iterable, Optional


DEFAULT_EXTENSIONS = {
    ".c", ".cc", ".cpp", ".cxx",
    ".h", ".hh", ".hpp", ".hxx",
    ".txt", ".md",
    ".cmake",
    ".ini", ".cfg", ".conf",
    ".json", ".xml", ".yml", ".yaml",
    ".rc", ".def",
    ".bat", ".cmd", ".sh",
    ".py",
}

SPECIAL_FILENAMES = {
    "CMakeLists.txt",
    "Makefile",
    "Dockerfile",
    ".editorconfig",
    ".gitattributes",
    ".gitignore",
}

SKIP_DIR_NAMES = {
    ".git",
    ".svn",
    ".hg",
    ".idea",
    ".vscode",
    "build",
    "out",
    "dist",
    "target",
    "node_modules",
    "__pycache__",
    ".encoding_fix_candidates",
    ".encoding_backups",
}

# 常见乱码片段。注意：单个字符可能误判，所以脚本不会只因为一个字符就直接覆盖。
MOJIBAKE_MARKERS = [
    "锟斤拷",
    "ï¿½",
    "\ufffd",

    # UTF-8 被 Latin-1 / Windows-1252 错读后常见
    "Ã",
    "Â",
    "â€",
    "â€œ",
    "â€�",
    "â€™",
    "ä¸",
    "ä½",
    "æ–",
    "æœ",
    "çš",
    "å",

    # UTF-8 被 GBK/CP936 错读后常见
    "涓",
    "枃",
    "鏂",
    "浠",
    "鎵",
    "搴",
    "鐮",
    "鍙",
    "绋",
    "缁",
    "妯",
    "璇",
    "閿",
    "堕",
    "杈",
    "寮",
    "侀",
    "忚",
]

PUA_RE = re.compile(r"[\ue000-\uf8ff]")


@dataclass
class Candidate:
    method: str
    text: str
    quality: int
    bad_score: int


@dataclass
class FileResult:
    path: Path
    status: str
    base_encoding: str
    base_quality: int
    base_bad_score: int
    best_method: str
    best_quality: int
    best_bad_score: int
    action: str
    candidate_path: str


def is_probably_binary(data: bytes) -> bool:
    if not data:
        return False

    sample = data[:4096]

    if b"\x00" in sample:
        return True

    # 控制字符比例过高，认为是二进制
    allowed = {9, 10, 13}  # tab, lf, cr
    control_count = sum(1 for b in sample if b < 32 and b not in allowed)
    return control_count / max(len(sample), 1) > 0.20


def should_process_file(path: Path, extensions: Optional[set[str]]) -> bool:
    if path.name in SPECIAL_FILENAMES:
        return True

    if extensions is None:
        return True

    return path.suffix.lower() in extensions


def is_skipped_path(path: Path) -> bool:
    return any(part in SKIP_DIR_NAMES for part in path.parts)


def decode_file_bytes(data: bytes) -> tuple[Optional[str], str]:
    """
    返回：
        text, encoding_name

    encoding_name 可能是：
        utf-8
        gb18030
        unknown
    """
    try:
        return data.decode("utf-8-sig"), "utf-8"
    except UnicodeDecodeError:
        pass

    try:
        return data.decode("gb18030"), "gb18030"
    except UnicodeDecodeError:
        pass

    return None, "unknown"


def count_cjk(text: str) -> int:
    return sum(1 for ch in text if "\u4e00" <= ch <= "\u9fff")


def count_pua(text: str) -> int:
    return len(PUA_RE.findall(text))


def count_bad_markers(text: str) -> int:
    score = 0

    # 替换字符非常危险，说明原信息可能已经损坏
    score += text.count("\ufffd") * 80

    # 私有区字符经常出现在 UTF-8 被 GBK 错读的场景
    score += count_pua(text) * 30

    for marker in MOJIBAKE_MARKERS:
        count = text.count(marker)
        if count:
            if marker in {"涓", "枃", "浠", "鎵", "鍙", "璇"}:
                score += count * 3
            else:
                score += count * 8

    return score


def count_printable(text: str) -> int:
    return sum(1 for ch in text if ch.isprintable() or ch in "\r\n\t")


def quality_score(text: str) -> int:
    """
    分数越高，越像正常文本。
    """
    cjk = count_cjk(text)
    bad = count_bad_markers(text)
    printable = count_printable(text)
    length = max(len(text), 1)

    printable_score = int(printable / length * 100)

    return cjk * 2 + printable_score - bad * 5


def looks_suspicious(text: str) -> bool:
    return count_bad_markers(text) >= 8


def try_transform(
    text: str,
    wrong_codec: str,
    correct_codec: str,
    errors: str = "strict",
) -> Optional[str]:
    """
    反向修复思路：

    当前 text 是乱码文本。
    如果它是“原字节被 wrong_codec 错误解码后得到的文本”，
    那么尝试先 encode 回 wrong_codec 的字节，
    再按 correct_codec 正确 decode。
    """
    try:
        return text.encode(wrong_codec, errors=errors).decode(correct_codec, errors=errors)
    except UnicodeError:
        return None
    except LookupError:
        return None


def build_transformers() -> list[tuple[str, Callable[[str], Optional[str]]]]:
    transformers: list[tuple[str, Callable[[str], Optional[str]]]] = []

    pairs = [
        # 原本 UTF-8，被 GBK/CP936/GB18030 错读
        ("gbk", "utf-8"),
        ("cp936", "utf-8"),
        ("gb18030", "utf-8"),

        # 原本 UTF-8，被 Latin-1 / CP1252 错读
        ("latin1", "utf-8"),
        ("cp1252", "utf-8"),

        # 原本 GBK/GB18030，被 Latin-1 / CP1252 错读
        ("latin1", "gb18030"),
        ("cp1252", "gb18030"),
    ]

    for wrong, correct in pairs:
        method = f"{correct}_misread_as_{wrong}"
        transformers.append(
            (
                method,
                lambda s, w=wrong, c=correct: try_transform(s, w, c, "strict"),
            )
        )

        # ignore 版本更激进，可能丢字符，所以只作为候选，不建议无脑覆盖
        method_ignore = f"{correct}_misread_as_{wrong}_ignore"
        transformers.append(
            (
                method_ignore,
                lambda s, w=wrong, c=correct: try_transform(s, w, c, "ignore"),
            )
        )

    return transformers


def generate_candidates(text: str, max_depth: int) -> list[Candidate]:
    transformers = build_transformers()

    seen: set[str] = {text}
    frontier: list[tuple[str, str, int]] = [("original", text, 0)]
    candidates: list[Candidate] = []

    while frontier:
        prev_method, current_text, depth = frontier.pop(0)

        if depth >= max_depth:
            continue

        for method, func in transformers:
            new_text = func(current_text)
            if not new_text:
                continue

            if new_text == current_text:
                continue

            if new_text in seen:
                continue

            seen.add(new_text)

            chained_method = method if prev_method == "original" else f"{prev_method}__then__{method}"

            cand = Candidate(
                method=chained_method,
                text=new_text,
                quality=quality_score(new_text),
                bad_score=count_bad_markers(new_text),
            )
            candidates.append(cand)

            frontier.append((chained_method, new_text, depth + 1))

    candidates.sort(key=lambda c: (c.quality, -c.bad_score), reverse=True)
    return candidates


def safe_candidate_filename(filename: str, method: str) -> str:
    safe_method = re.sub(r"[^A-Za-z0-9_.-]+", "_", method)
    safe_method = safe_method[:100]
    return f"{filename}.{safe_method}.candidate"


def write_utf8(path: Path, text: str) -> None:
    path.parent.mkdir(parents=True, exist_ok=True)
    path.write_bytes(text.encode("utf-8"))


def backup_original(root: Path, path: Path, backup_dir: Path) -> Path:
    rel = path.relative_to(root)
    backup_path = backup_dir / rel
    backup_path.parent.mkdir(parents=True, exist_ok=True)
    shutil.copy2(path, backup_path)
    return backup_path


def process_file(
    root: Path,
    path: Path,
    candidate_dir: Path,
    backup_dir: Path,
    apply: bool,
    min_improvement: int,
    max_depth: int,
) -> FileResult:
    rel = path.relative_to(root)

    data = path.read_bytes()

    if is_probably_binary(data):
        return FileResult(
            path=rel,
            status="BINARY_SKIPPED",
            base_encoding="binary",
            base_quality=0,
            base_bad_score=0,
            best_method="",
            best_quality=0,
            best_bad_score=0,
            action="SKIPPED",
            candidate_path="",
        )

    text, encoding = decode_file_bytes(data)

    if text is None:
        return FileResult(
            path=rel,
            status="UNKNOWN_ENCODING",
            base_encoding="unknown",
            base_quality=0,
            base_bad_score=0,
            best_method="",
            best_quality=0,
            best_bad_score=0,
            action="MANUAL_CHECK",
            candidate_path="",
        )

    base_quality = quality_score(text)
    base_bad = count_bad_markers(text)

    candidates: list[Candidate] = []

    # 非 UTF-8 但可按 GB18030 读取：优先生成“转 UTF-8”的候选
    if encoding == "gb18030":
        candidates.append(
            Candidate(
                method="convert_gb18030_to_utf8",
                text=text,
                quality=quality_score(text),
                bad_score=count_bad_markers(text),
            )
        )

    # 疑似乱码时，再生成反乱码候选
    if looks_suspicious(text):
        candidates.extend(generate_candidates(text, max_depth=max_depth))

    # UTF-8 且不疑似乱码
    if not candidates:
        return FileResult(
            path=rel,
            status="NORMAL_UTF8" if encoding == "utf-8" else "NORMAL_TEXT",
            base_encoding=encoding,
            base_quality=base_quality,
            base_bad_score=base_bad,
            best_method="",
            best_quality=base_quality,
            best_bad_score=base_bad,
            action="NO_CHANGE",
            candidate_path="",
        )

    candidates.sort(key=lambda c: (c.quality, -c.bad_score), reverse=True)
    best = candidates[0]

    improvement = best.quality - base_quality

    should_apply = False
    status = "CANDIDATE_GENERATED"
    action = "REVIEW_CANDIDATE"

    # GB18030 正常文本转 UTF-8：属于编码规范化，可自动应用
    if best.method == "convert_gb18030_to_utf8":
        should_apply = apply
        status = "GB18030_TEXT_NEEDS_UTF8_NORMALIZE"
        action = "APPLIED" if apply else "REVIEW_CANDIDATE"

    # 疑似乱码修复：只有候选明显更好才允许 apply
    elif improvement >= min_improvement and best.bad_score < base_bad:
        should_apply = apply
        status = "SUSPECT_MOJIBAKE"
        action = "APPLIED" if apply else "REVIEW_CANDIDATE"
    else:
        status = "SUSPECT_BUT_NO_SAFE_FIX"
        action = "MANUAL_CHECK"

    candidate_path = candidate_dir / rel.parent / safe_candidate_filename(rel.name, best.method)
    write_utf8(candidate_path, best.text)

    if should_apply:
        backup_original(root, path, backup_dir)
        write_utf8(path, best.text)

    return FileResult(
        path=rel,
        status=status,
        base_encoding=encoding,
        base_quality=base_quality,
        base_bad_score=base_bad,
        best_method=best.method,
        best_quality=best.quality,
        best_bad_score=best.bad_score,
        action=action,
        candidate_path=str(candidate_path.relative_to(root)),
    )


def iter_target_files(root: Path, extensions: Optional[set[str]]) -> Iterable[Path]:
    for path in root.rglob("*"):
        if not path.is_file():
            continue

        rel = path.relative_to(root)

        if is_skipped_path(rel):
            continue

        if should_process_file(path, extensions):
            yield path


def parse_extensions(raw: str) -> Optional[set[str]]:
    raw = raw.strip()

    if raw in {"*", "all", "ALL"}:
        return None

    result = set()
    for item in raw.split(","):
        ext = item.strip().lower()
        if not ext:
            continue
        if not ext.startswith("."):
            ext = "." + ext
        result.add(ext)

    return result


def main() -> int:
    parser = argparse.ArgumentParser(
        description="扫描并修复大型项目中的中文乱码问题。默认只生成候选文件，不覆盖原文件。"
    )

    parser.add_argument(
        "--root",
        default=".",
        help="项目根目录，默认当前目录。",
    )

    parser.add_argument(
        "--apply",
        action="store_true",
        help="确认后覆盖原文件。覆盖前会备份到 .encoding_backups。",
    )

    parser.add_argument(
        "--extensions",
        default=",".join(sorted(DEFAULT_EXTENSIONS)),
        help="要处理的文件扩展名，逗号分隔。传 all 表示处理所有文件。",
    )

    parser.add_argument(
        "--candidate-dir",
        default=".encoding_fix_candidates",
        help="候选修复文件输出目录。",
    )

    parser.add_argument(
        "--backup-dir",
        default=".encoding_backups",
        help="apply 覆盖前的原文件备份目录。",
    )

    parser.add_argument(
        "--report",
        default="encoding_fix_report.csv",
        help="扫描报告 CSV 文件路径。",
    )

    parser.add_argument(
        "--min-improvement",
        type=int,
        default=30,
        help="候选修复质量至少提升多少才允许自动 apply。默认 30。",
    )

    parser.add_argument(
        "--max-depth",
        type=int,
        default=1,
        choices=[1, 2],
        help="反乱码尝试深度。1 较保守，2 可尝试二次乱码修复。",
    )

    args = parser.parse_args()

    root = Path(args.root).resolve()
    candidate_dir = root / args.candidate_dir
    backup_dir = root / args.backup_dir
    report_path = root / args.report

    extensions = parse_extensions(args.extensions)

    results: list[FileResult] = []

    for path in iter_target_files(root, extensions):
        try:
            result = process_file(
                root=root,
                path=path,
                candidate_dir=candidate_dir,
                backup_dir=backup_dir,
                apply=args.apply,
                min_improvement=args.min_improvement,
                max_depth=args.max_depth,
            )
            results.append(result)
        except Exception as exc:
            rel = path.relative_to(root)
            results.append(
                FileResult(
                    path=rel,
                    status="ERROR",
                    base_encoding="",
                    base_quality=0,
                    base_bad_score=0,
                    best_method="",
                    best_quality=0,
                    best_bad_score=0,
                    action=f"ERROR: {exc}",
                    candidate_path="",
                )
            )

    with report_path.open("w", encoding="utf-8-sig", newline="") as f:
        writer = csv.writer(f)
        writer.writerow(
            [
                "path",
                "status",
                "base_encoding",
                "base_quality",
                "base_bad_score",
                "best_method",
                "best_quality",
                "best_bad_score",
                "action",
                "candidate_path",
            ]
        )

        for r in results:
            writer.writerow(
                [
                    str(r.path),
                    r.status,
                    r.base_encoding,
                    r.base_quality,
                    r.base_bad_score,
                    r.best_method,
                    r.best_quality,
                    r.best_bad_score,
                    r.action,
                    r.candidate_path,
                ]
            )

    total = len(results)
    review = sum(1 for r in results if r.action == "REVIEW_CANDIDATE")
    applied = sum(1 for r in results if r.action == "APPLIED")
    manual = sum(1 for r in results if r.action == "MANUAL_CHECK")
    skipped = sum(1 for r in results if r.action == "SKIPPED")
    errors = sum(1 for r in results if r.status == "ERROR")

    print(f"扫描完成：{total} 个文件")
    print(f"需要人工检查候选：{review}")
    print(f"已自动覆盖修复：{applied}")
    print(f"需要人工处理：{manual}")
    print(f"跳过文件：{skipped}")
    print(f"错误文件：{errors}")
    print(f"报告文件：{report_path}")

    if not args.apply:
        print()
        print("当前为 dry-run 模式，未覆盖原文件。")
        print(f"候选修复文件目录：{candidate_dir}")
        print("确认候选文件正常后，可重新运行并添加 --apply。")

    return 0


if __name__ == "__main__":
    raise SystemExit(main())
```

## 6. 使用方法

### 6.1 第一次只扫描，不覆盖

在项目根目录执行：

```bash
python tools/fix_chinese_encoding.py --root .
```

脚本会生成：

```text
encoding_fix_report.csv
.encoding_fix_candidates/
```

其中：

- `encoding_fix_report.csv` 是扫描报告；
- `.encoding_fix_candidates/` 是候选修复文件目录；
- 原始文件不会被覆盖。

---

### 6.2 查看报告

打开：

```text
encoding_fix_report.csv
```

重点看以下列：

| 列名           | 含义                     |
| -------------- | ------------------------ |
| path           | 原文件路径               |
| status         | 文件状态                 |
| base_encoding  | 当前识别出的原始编码     |
| base_bad_score | 原文件乱码可疑程度       |
| best_method    | 推荐修复方法             |
| best_bad_score | 修复候选文件乱码可疑程度 |
| action         | 建议动作                 |
| candidate_path | 候选修复文件路径         |

常见状态说明：

| status                            | 含义                                 |
| --------------------------------- | ------------------------------------ |
| NORMAL_UTF8                       | 正常 UTF-8 文件                      |
| GB18030_TEXT_NEEDS_UTF8_NORMALIZE | 文件可能是 GBK/GB18030，需要转 UTF-8 |
| SUSPECT_MOJIBAKE                  | 疑似可修复乱码                       |
| SUSPECT_BUT_NO_SAFE_FIX           | 疑似乱码，但没有安全候选             |
| UNKNOWN_ENCODING                  | 无法识别编码，需要人工处理           |
| BINARY_SKIPPED                    | 二进制文件，已跳过                   |

---

### 6.3 人工检查候选文件

例如报告中显示：

```text
path: src/main.cpp
candidate_path: .encoding_fix_candidates/src/main.cpp.utf-8_misread_as_gbk.candidate
```

可以用 VS Code 对比：

```bash
code --diff src/main.cpp .encoding_fix_candidates/src/main.cpp.utf-8_misread_as_gbk.candidate
```

或者直接打开候选文件看中文是否恢复正常。

---

### 6.4 确认后自动覆盖

确认候选修复结果整体正常后，再执行：

```bash
python tools/fix_chinese_encoding.py --root . --apply
```

脚本会：

1. 把原文件备份到 `.encoding_backups/`；
2. 用候选修复内容覆盖原文件；
3. 重新生成 `encoding_fix_report.csv`。

---

### 6.5 尝试二次乱码修复

如果项目经历过多次错误编码保存，可以尝试：

```bash
python tools/fix_chinese_encoding.py --root . --max-depth 2
```

确认候选文件无误后再执行：

```bash
python tools/fix_chinese_encoding.py --root . --max-depth 2 --apply
```

注意：`--max-depth 2` 更激进，必须认真检查候选文件。

## 10. 避免再次出现乱码

### 10.1 增加 `.editorconfig`

在项目根目录新建 `.editorconfig`：

```ini
root = true

[*]
charset = utf-8
end_of_line = lf
insert_final_newline = true
trim_trailing_whitespace = true

[*.{c,cc,cpp,cxx,h,hh,hpp,hxx}]
charset = utf-8

[*.{txt,md,cmake,ini,cfg,json,xml,yml,yaml}]
charset = utf-8
```

---

### 10.2 增加 `.gitattributes`

在项目根目录新建 `.gitattributes`：

```gitattributes
* text=auto

*.c     text
*.cc    text
*.cpp   text
*.cxx   text
*.h     text
*.hh    text
*.hpp   text
*.hxx   text
*.txt   text
*.md    text
*.cmake text
*.json  text
*.xml   text
*.yml   text
*.yaml  text
```

`.gitattributes` 主要用于规范文本文件处理和换行行为。
编码统一仍然主要依赖 IDE 配置、`.editorconfig` 和团队约定。

---

### 10.3 配置 Git 中文路径显示

如果中文文件名在 Git 中显示成：

```text
"\346\226\207\344\273\266.cpp"
```

执行：

```bash
git config --global core.quotepath false
```

### 10.4 CMake 中为 MSVC 增加 UTF-8 编译选项

如果使用 Visual Studio / MSVC 编译器，建议在 `CMakeLists.txt` 中加入：

```cmake
if(MSVC)
    add_compile_options(/utf-8)
endif()
```

完整示例：

```cmake
cmake_minimum_required(VERSION 3.20)
project(MyProject)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

if(MSVC)
    add_compile_options(/utf-8)
endif()

add_executable(MyProject main.cpp)
```

这样可以让 MSVC 按 UTF-8 读取源文件，减少中文字符串和注释相关问题。

## 11. 重要结论

远程仓库网页端已经乱码时，说明当前仓库里的文件大概率已经被错误保存过。
此时本地 clone 后再切换 GBK、UTF-8 打开，通常无法解决。

正确处理方式是：

```text
先保护现场
再生成候选修复
再人工确认
最后批量覆盖
```

脚本只能修复“可逆乱码”。
如果内容已经变成大量 `���` 或 `?????`，原始信息可能已经丢失，需要从历史版本、备份、队友本地代码或人工重写恢复。
